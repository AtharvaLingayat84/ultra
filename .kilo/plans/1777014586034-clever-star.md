# Plan: Fix receiver decode scope and validate build environment

## Goal
Make `Start Listening` decode a full ultrasonic frame reliably, and verify the Android build environment on the other machine so the fix can actually be tested there.

## Known blocker
The receiver currently decodes only the newest 3 seconds of audio, but the framed transmission is longer than that. `Decoder.decodeBits()` therefore often never sees both `START_MARKER` and `END_MARKER` in the same stream.

## Implementation plan
1. **Expand receive scope**
   - Update `Receiver.kt` so decoding uses the full rolling buffer instead of only the newest 3-second slice.
   - Keep the rolling buffer, tail trimming, and cooldown logic intact.
   - Preserve the sliding demodulation scan.

2. **Add explicit decode diagnostics**
   - Update `Decoder.kt` to log when:
     - the bitstream is empty
     - `START_MARKER` is missing
     - `END_MARKER` is missing after `START_MARKER`
     - a frame is successfully extracted
   - Log payload length and decoded character count.

3. **Keep transport protocol unchanged**
   - Do not alter FSK, repeat bits, or marker format.
   - Do not change the audible test sweep path.

4. **Size the receive buffer appropriately**
   - Ensure receive retention is long enough to hold a full framed message for the supported settings.
   - Keep duplicate suppression behavior after success.

5. **Validate on the other PC**
   - Confirm the Android/JDK/Gradle setup is usable before running tests.
   - Verify:
     - `java -version`
     - the project Gradle wrapper works (`./gradlew` / `gradlew.bat`)
     - the configured JDK matches the Android Gradle Plugin requirements
   - If Gradle fails, inspect:
     - `gradle.properties`
     - `build.gradle` / `build.gradle.kts`
     - `settings.gradle` / `settings.gradle.kts`
     - `JAVA_HOME`
     - `org.gradle.java.home`

## Success criteria
- `Start Listening` decodes a real framed message.
- Logs clearly explain marker presence and rejection reasons.
- Repeated sends do not create duplicate messages.
- Audible test sweep still works.
- The other machine can build the app with a valid JDK/Gradle setup.
