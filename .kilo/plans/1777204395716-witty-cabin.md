# Plan: Crypto Terminal UI Refresh

## Goal
Redesign the Android app UI to feel modern, dark, technical, and crypto-terminal inspired while keeping the existing ultrasonic send/receive functionality unchanged.

## Scope Assumption
- Stay on the current Android XML/View-based setup.
- Preserve all current actions: send, receive, sweep, tuning inputs, logs.
- Change visual language, layout hierarchy, spacing, and resource styling only.

## Plan
1. Replace the current light AppCompat theme with a dark, high-contrast terminal palette.
2. Introduce a compact visual system:
   - near-black background
   - neon green / cyan accent colors
   - monospace for logs and technical values
   - subtle borders, glow-like accents, and rounded panels
3. Rebuild `activity_main.xml` into a dashboard-style layout:
   - top status header with app title and live status
   - primary message composer card
   - tuning controls grouped as technical settings
   - action row for send/receive/sweep
   - dedicated console/log panel with terminal feel
4. Update text, labels, and hierarchy to sound more technical and crypto-native where appropriate.
5. Add reusable drawables or shape resources if needed for panel backgrounds, borders, and button states.
6. Verify responsive behavior on small and large screens:
   - no horizontal overflow
   - readable at mobile sizes
   - clear touch targets
7. Validate accessibility and polish:
   - contrast stays readable in dark mode
   - focus states remain visible
   - logs remain selectable and easy to scan

## Visual Direction
- Overall mood: crypto trading terminal, command console, encrypted signal dashboard.
- Avoid generic Material look.
- Prefer dense but organized panels, technical labels, and restrained neon accents.
- Use motion sparingly if added later; keep the surface mostly static and sharp.

## File Targets
- `app/src/main/res/values/themes.xml`
- `app/src/main/res/values/colors.xml`
- `app/src/main/res/values/strings.xml`
- `app/src/main/res/layout/activity_main.xml`
- optional drawable resources under `app/src/main/res/drawable/`

## Acceptance Criteria
- The app reads as a dark crypto-terminal interface, not a default form screen.
- Core controls are still obvious and usable.
- Logs look like a console output area.
- UI remains clean on phones and tablets.

## Open Question
- If you want, I can make it fully dark-only, or keep a compatible light theme fallback for system theme support.
